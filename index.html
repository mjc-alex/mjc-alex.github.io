PA2-1实验报告

指令模拟，基本设置如下：

  • make run or make test_pa-2-1编译项目，得到测试用例的可执行文件和二进制镜像文件。
  • ./nemu/nemu --autorun --testcase add, NEMU执行测试用例add
  • exec_inst()返回指令长度，也在cpu.c里，在函数体中，模拟执行指令，opcode_entry一个函数数组，执行对应的函数（指令）

关于指令模拟

  • 指令格式

  • 若opcode前面出现0x66，则操作数长度临时改变为16位，否则为32位

  • /nemu/src/cpu/inst里有opcode对应的函数，比如make_instr_func(mov_i2rm_v)，这里有一层宏展开，展开为：mov_i2rm_v(uint32_t eip, uint8_t opcode).推荐函数命名规则：指令名_源操作数类型2目的操作数类型_长度后缀。在/nemu/src/cpu/instr里对应的.c文件补充实现。
    src_type和dest_type是源和目的操作数类型，与decode_operand系列宏一致：
    rm – 寄存器或内存地址 – 对应手册E类型
    r – 寄存器地址 – 对应手册G类型
    i – 立即数 – 对应手册I类型
    m – 内存地址 – 差不多对应手册M类型
    a – 根据操作数长度对应al, ax, eax – 手册里没有
    c – 根据操作数长度对应cl, cx, ecx – 手册里没有
    o – 偏移量 – 对应手册里的O类型
    suffix是操作数长度后缀，与decode_data_size系列宏一致：
    b, w, l, v – 8，16，32，16/32位
    bv – 源操作数为8位，目的操作数为16/32位，特殊指令用到
    short, near – jmp指令用到，分别指代8位和32位

  • 综上，创建一个指令需要：

    • 在nemu/src/cpu/instr/里补充函数实现
    • 创建头文件nemu/include/cpu/instr/mov.h，并在里面写入以下内容并保存：
    • 编辑nemu/include/cpu/instr.h头文件，在里面include上述头文件：#include "cpu/instr/mov.h"
    • 在opcode_entry[]数组中对应的位置填入该函数的名称

  • 适用宏的指令：adc, add, and, bt, cbw, cmov, cmp, dec, inc, jcc, 大多数的mov, not, or, pop, push, sar, sbb, setcc, shl, shr, sub, test, xor

  • 不适用的指令：call, cltd, cmps, div, idiv, mul, imul, cld, clc, sahf, hlt, int, jmp, lea, leave, rep, ret, stos, x87

ICS note

PA 结构：

汇编相关

  • 操作数后缀为l为双字，32位，为w为单字，16位。省略默认32位。objdump and gcc use AT&T assembly language. AT&T’s source operand is on the left and the destination operand is on the right. The format of assembly language “ op src dst” means “dst <– src op dst”. “100(%ebx, %esi, 4)” in “MOV “ instruction means M[R[ebx] + 4 * R[esi] + 100].
    However, leal (%eax,%ebx), %eax means R[eax] + R[ebx] = R[eax]
  • IA-32’s registers has 32 bits, if just given a b or w, it will use 8 or 16 lower bits.
  • somes instructions :
  1. PUSH: 先执行R[sp] <– R[sp] - 2 or R[esp] <– R[esp] - 4. 然后将一个字或者双字送入sp or esp 指示的栈单元。pushw means push a single word into stack and pushl means push double words.
  2. POP: 先将一个字或者双字从sp or esp指示的栈单元送入指定寄存器，再执行R[sp] <– R[sp] + 2 or R[esp] <– R[esp] + 4
  3. LEA: 必须是存储器寻址方式，比如基址加变址寄存器乘比例系数加偏移。目的寄存器不能是段寄存器。
  4. MOV: 一定注意，这个指令的两个操作数不能同时为存储器操作数。因为同一时刻，CPU对RAM只有读和写的两种操作之一，在汇编语言的底层实现上，不能直接对RAM的两个数据操作，因此使用一个寄存器用来中转数据
  5. CMP：目的操作数减去源操作数
  6. ADD/SUB: 两个操作数中至多有一个存储器操作数，不区分无符号数还是带符号整数
  7. INC/DEC: 自增/自减，注意不生成CF标志位
  8. NEG: 取负指令，也即各位取反，末尾加一，注意若字节操作数为-128或者字操作数为-32768，则操作数无变化，但是OF=1。若是操作数为0，CF = 0，否则其他情况CF = 1。
  9. NOT/OR/AND/XOR: 改变目的操作数，OF = CF = 0，ZF 和SF看情况
  10. TEST: 两个操作数相与，但是不改变操作数，设置相应标志位

IA-32架构

  • How registers are organized :
    8 general-purpose registers: eax, ebx ecx, edx. They are used to store operands. esp, ebp, esi, edi are mostly used to store offset values or pointers, and esp is stack pointer register, ebp is base address pointer register.
    2 dedicated registers : eip and eflags. eip expands from 16-bits ip, which is used exactly the same as pc register.
  • gcc规定每个函数的栈帧必须是16字节的倍数
  • IA-32中，四个寄存器EBP, ESP,EDI,ESI的低8位不能使用

关于i386手册

  • opcode map: Use the high-order four bits of the opcode as an index to a row of the opcode
    table; use the low-order four bits as an index to a column of the table.
  • The ModR/M byte contains three fields of information:
    • The mod field, which occupies the two most significant bits of the byte, combines with the r/m field to form 32 possible values: eight registers and 24 indexing modes
    • The reg field, which occupies the next three bits following the mod field, specifies either a register number or three more bits of opcode information. The meaning of the reg field is determined by the first (opcode) byte of the instruction.
    • The r/m field, which occupies the three least significant bits of the byte, can specify a register as the location of an operand, or can form part of the addressing-mode encoding in combination with the field as described above
      The based indexed and scaled indexed forms of 32-bit addressing require the SIB byte. The presence of the SIB byte is indicated by certain encodings of the ModR/M byte.
  • The SIB byte then includes the following fields:
    • The ss field, which occupies the two most significant bits of the byte, specifies the scale factor
    • The index field, which occupies the next three bits following the ss field and specifies the register number of the base register
    • The base field, which occupies the three least significant bits of the byte, specifies the register number of the base register

PA2-1

指令模拟，基本设置如下：

  • make run or make test_pa-2-1编译项目，得到测试用例的可执行文件和二进制镜像文件。
  • ./nemu/nemu --autorun --testcase add, NEMU执行测试用例add
  • exec_inst()返回指令长度，也在cpu.c里，在函数体中，模拟执行指令，opcode_entry一个函数数组，执行对应的函数（指令）

关于指令模拟

  • 指令格式

  • 若opcode前面出现0x66，则操作数长度临时改变为16位，否则为32位

  • /nemu/src/cpu/inst里有opcode对应的函数，比如make_instr_func(mov_i2rm_v)，这里有一层宏展开，展开为：mov_i2rm_v(uint32_t eip, uint8_t opcode).推荐函数命名规则：指令名_源操作数类型2目的操作数类型_长度后缀。在/nemu/src/cpu/instr里对应的.c文件补充实现。

关于i386手册

  • opcode map: Use the high-order four bits of the opcode as an index to a row of the opcode
    table; use the low-order four bits as an index to a column of the table.
  • 几种jump:
    • Near jump—A jump to an instruction within the current code segment (the segment currently pointed to by the CS register), sometimes referred to as an intrasegment jump.
    • Short jump—A near jump where the jump range is limited to –128 to +127 from the current EIP value.
    • Far jump—A jump to an instruction located in a different segment than the current code segment but at the same privilege level, sometimes referred to as an intersegment jump
